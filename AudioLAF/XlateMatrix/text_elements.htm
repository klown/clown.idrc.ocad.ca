"Non-Sonified" Text Components

 

Description

All text-based components share certain common attributes, by virtue of having a common ancestor. These attributes will be discussed first, then the specific text-based components will be introduced, and the specifics of each component will be explained.

    Text

    Obviously, text-based components have some form of text area that allows users to enter and view text. Basic text area functionality included is:

      cursor up/down/left/right
      cursor home/end
      backspace
      delete

    Caret

    Indicates the current insertion point.

    Selected Text

    Text that has been selected by dragging a mouse over it, or by arrowing over it with the shift key depressed.

    Disabled Text

    I'm not sure what this is. It is likely text which cannot be editted (or selected?).

    Highlight

    This is something that is applied to the background of the text area to distinguish it from the rest of the area. It is not the same as selecting, in that it does not identify any of the text, but rather a region of the area.

"Lack of Augmented Sound" Rationale

These functions are related to the ability to rapidly edit text. In this process, a computer needs to have instant feedback in text. Adding individual sounds (bells, beeps, bongs) to functions only delays the editing process and frustrates the computer user. In terms of Navigating Within, Speaking Actions, Text Selection and Text Actions, if a computer user is proficient additional sound would interfere with the thought process and this would be true if the computer user were a novice as well, the sound would detract from listening to the functional feedback.
