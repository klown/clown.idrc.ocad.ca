Resulting Accessibles

With an @aria-label, the declared name property is different from the actual content of the list item, here, "winking smile" vs. the smiley text ";-)". Without an @aria-label, there is no declared name, and only content.

  • FF: With @aria-label, exposes "winking smile" as the name, and the smiley text as the content. Without, the name and content are both the smiley text.
  • IE9: With, the name is "winking smile", and the text child is the smiley text. Without, the name and the text child are both the smiley text.
  • Safari/OSX: With or without, there is no name and only an AXValue containing the smiley text.
